## Conventions to follow

- **Background filename = `background_prompt_key`** (no world prefix, no
  `_NNN`). E.g. `ruined_courtyard_dawn.png`, `polyphemus_cave.png`.
  All in flat folders under `assets/backgrounds/<world>/`.
- **Figure compositing is alpha-gated.** `render.py` skips opaque figure
  assets — the AI image is treated as the final scene. To composite a
  figure layer, the asset must be a transparent PNG. (Currently all
  figure files are RGB, so all are skipped.)
- **MP4 named after folder**: `outputs/drafts/<id>/<id>.mp4`.
- **Six motion profiles** in `render.py:_MOTION_PARAMS`: `static`,
  `slow_zoom`, `slow_pan`, `medium_zoom`, `medium_pan`, `figure_pan`.
  All use a single PIL AFFINE transform per frame — sub-pixel correct,
  no integer-rounding stepping. **Trade-off**: each render takes ~4 min
  (not 30s) due to the Python-level per-frame compositing.
- **Source entries** live in `content_library/sources/<author>/<work>.md`
  as YAML blocks separated by `---`. Schema in
  `content_library/01_SCHEMA.md`. **Every entry has
  `translation_check_needed: true`** — verify against the cited edition
  before publishing.
- **Arc YAMLs** live in `content_library/arcs/<region>_<arc>.yaml`.
  See `07_ARCS_AND_PRODUCTION.md`.
- **Per-video script YAMLs** live in
  `content_library/scripts/<region>/<arc>/NNN_<slug>.yaml`. ~65–85
  words per ~30-second video.
- **Pipeline-format specs** (the format `motive render` consumes) live
  in `outputs/specs/<region>/<arc>/<spec_id>.yaml`. Generated by
  `scripts/generate_ancient_greece_drafts.py` from the content_library
  scripts. **Re-run that transformer** when you add a new ancient_greece
  script.

## Gotchas / things I learned the hard way

- **ComfyUI must be running on the right port.** If
  `curl http://127.0.0.1:8000` returns nothing, none of `make-images`
  will work. The config used to be 8188 — current is 8000.
- **The workflow JSON must be valid.** The placeholder shipped with
  `ckpt_name: REPLACE_WITH_YOUR_SDXL_CHECKPOINT.safetensors` — the
  current file points at `Juggernaut-XL_v9_RunDiffusionPhoto_v2.safetensors`.
  If you regenerate it, make sure the six tokens are still in place and
  the `__SEED__`/`__WIDTH__`/`__HEIGHT__` tokens are **unquoted** (bare
  number slots).
- **PIL-affine motion is slow.** Each render is ~4 minutes. Plan batches
  accordingly. A potential speedup path is switching to NumPy or OpenCV
  for the per-frame affine, but it isn't done yet.
- **Don't double-batch.** If a `motive render` batch is running and
  you start another one, both will write to the same draft directories.
  Kill `motive.exe` and `ffmpeg.exe` via `taskkill` before relaunching.
- **Old asset filenames had world prefixes** (e.g.
  `mythic_polyphemus_cave_001.png`). They've been renamed to simple keys
  (`polyphemus_cave.png`) to match the convention the rest of the project
  uses. Don't reintroduce the prefix.
- **Tests live in `tests/`. Run with `.venv/Scripts/python.exe -m pytest -q`.**
  117/117 should pass; if you add code, mirror existing patterns.
- **The transformer at `scripts/generate_ancient_greece_drafts.py`** is
  how content_library scripts become outputs/specs YAMLs. When you add
  a new ancient_greece script, add its row to the `SCRIPTS` list and
  re-run the transformer.
- **For specs whose background is in place**, set `bg_source: None` in
  the transformer entry — it skips the asset copy and looks for an
  existing file at `assets/backgrounds/<world>/<bg_key>.<ext>`.

## How to launch the review UI

```
.venv/Scripts/motive.exe review --no-browser --port 8501
```

then open `http://localhost:8501` in a browser. It lists every draft in
`outputs/drafts/`, plays the MP4, and lets you route to
`outputs/approved/` or `outputs/rejected/`.

## Ask before doing

- **Anything destructive**: deleting source entries, deleting drafts,
  force-pushing, `--no-verify`, dropping tables. Confirm first.
- **Big structural changes**: renaming arcs, renaming worlds, changing
  the spec schema. The choreography and transformer reference these by
  name.
- **New ComfyUI workflows**: keep the six tokens in place; numeric
  tokens (`__SEED__`, `__WIDTH__`, `__HEIGHT__`) go in bare number slots
  (no surrounding quotes), string tokens (`__POSITIVE__`, `__NEGATIVE__`,
  `__FILENAME_PREFIX__`) go inside `"..."`.

When you start, glance at `git status` to see what's uncommitted, then
ask the user what they want to work on. The roadmap's priority section
is a strong default.

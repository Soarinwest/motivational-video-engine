# 04 — Agent Workflows

> Deliberately empty. There is no agent framework in motive-engine today, and adding one is not on the MVP roadmap.

## Why no agents (yet)

The MVP pipeline is deterministic and operator-driven:

- **Scripts** are hand-authored in YAML. No LLM in the loop.
- **Visual assets** are generated by the operator in their tool of choice (ComfyUI, Krea, Midjourney, local SD), guided by prompts the engine emits.
- **TTS, captions, render** are deterministic local code.
- **Review** is a human watching a video.

A multi-agent orchestration layer would add machinery and indirection without solving a problem the MVP has.

## Where an "agent" could live later

If the engine ever grows an LLM touchpoint, the obvious candidates are:

1. **YAML drafting assistance.** Turn a one-line idea into a draft `ContentSpec` for the operator to refine.
2. **Editorial second opinion.** Score a draft script against [01_CONTENT_PHILOSOPHY.md](01_CONTENT_PHILOSOPHY.md) and flag drift.
3. **Prompt refinement.** Rewrite a visual prompt for a specific image generator.

Each of these is a *call-and-return* function, not an agent loop. None require LangChain, AutoGen, CrewAI, or similar.

## Rule

If we ever add an LLM call, it must be:

- **Local-preferred** (Ollama, llama.cpp, LM Studio) — same local-first principle as the rest of the engine.
- **Direct** — a single function, not a framework.
- **Optional** — the engine must run without it.
